# 邻里互助管理端

邻里互助管理端服务于运营、审核、风控、财务、客服等后台角色，所有页面以真实接口、真实字段、真实状态和真实权限为交付标准。

## 项目定位

- 平台名称：邻里互助 / LinBang
- 管理端接口前缀：`/admin-api/*`
- 用户端接口前缀：`/app-api/*`
- 默认单平台模式：前端不展示租户切换，租户号固定按 `1` 传递
- 地图策略：当前管理端未启用地图拾取能力；后续如接入地图，仅允许使用高德

## 技术栈

- Vue 3
- Vite
- TypeScript
- Element Plus
- Pinia

## 本地联调基线

- 后端地址：`http://127.0.0.1:48080`
- 数据库：`linbang`
- Redis：`127.0.0.1:6379`

## 安装依赖

要求：

- Node.js `>= 20.19.0`
- `pnpm >= 8.6.0`

安装：

```bash
pnpm install
```

## 环境切换与启动

管理端通过 Vite `mode` 切换环境，命令和环境文件一一对应：

| 命令 | 对应环境文件 | 用途 |
| --- | --- | --- |
| `pnpm dev` | `.env.local` | 本机前后端一起联调，默认开发方式 |
| `pnpm dev-server` | `.env.dev` | 本机只启前端，连接单独的开发后端 |
| `pnpm build:local` | `.env.local` | 本地联调打包 |
| `pnpm build:dev` | `.env.dev` | 开发环境打包 |
| `pnpm build:test` | `.env.test` | 测试环境打包 |
| `pnpm build:stage` | `.env.stage` | 预发布环境打包 |
| `pnpm build:prod` | `.env.prod` | 生产环境打包 |

默认本地启动：

```bash
pnpm dev
```

默认访问地址：

- `http://127.0.0.1:80`

如果端口被占用，修改根目录 `.env` 中的 `VITE_PORT` 后重新启动。

## 常见问题

### 1. `ERR_PNPM_IGNORED_BUILDS`

如果使用 `pnpm 11`，首次安装或打包时可能看到以下报错：

```bash
ERR_PNPM_IGNORED_BUILDS
```

这是 `pnpm` 的依赖脚本安全策略导致的，不是项目代码报错。执行一次下面的命令即可：

```bash
pnpm approve-builds --all
pnpm install
```

处理完成后，再执行：

```bash
pnpm dev
```

或：

```bash
pnpm build:local
```

## 环境文件说明

- `.env`：全局公共配置，例如标题、端口、验证码开关、默认登录账号
- `.env.local`：本地联调环境，默认连接 `127.0.0.1:48080`
- `.env.dev`：开发环境配置
- `.env.test`：测试环境打包配置
- `.env.stage`：预发布环境打包配置
- `.env.prod`：生产环境打包配置

切换到非本地环境时，优先修改对应环境文件中的以下字段：

- `VITE_BASE_URL`：后端服务地址
- `VITE_API_URL`：接口前缀，当前固定为 `/admin-api`
- `VITE_MALL_H5_DOMAIN`：后续用户端 H5 域名占位
- `VITE_GOVIEW_URL`：大屏或报表服务地址占位
- `VITE_BASE_PATH`：部署子路径

## 当前业务范围

- 用户中心：用户、地址、实名认证、资质审核
- 服务商中心：入驻审核、服务类目、服务点、服务商资料
- 订单中心：主订单、拆分单元、异常单、锁单解锁、抢单记录
- 资金中心：支付订单、退款审核、钱包、提现、银行卡、分账规则
- 风控中心：风控规则、敏感词、风险事件
- 售后与信用：投诉、申诉、评价、信用规则
- 运营协同：消息、推广、合作商、数据中心

## 开发约束

- 仅保留邻里互助业务相关菜单、页面、文案与操作流
- 不展示脚手架品牌词、演示入口、演示统计脚本
- 长期 mock 数据、空按钮、假动作页面不视为完成
- 状态字典、字段契约、接口命名以仓库 `spec/` 规则文件为准
- 支付、退款、提现、审核、风控必须与后端真实状态联动
