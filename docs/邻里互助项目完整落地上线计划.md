# 邻里互助项目完整落地上线计划

## 1. 文档目的

本文件作为邻里互助项目的统一落地执行计划，用于：

1. 指挥后端、管理端、联调、测试、上线全过程
2. 作为跨会话、跨阶段的统一进度面板
3. 确保每次开发都围绕真实接口、真实表结构、真实业务规则推进
4. 最终把项目推进到“可上线”状态，而不是只停留在原型、壳页面或局部模块

## 2. 使用规则

后续所有会话推进时，必须遵守：

1. 开工前先读取 [AGENTS.md](../AGENTS.md)
2. 规则以 [business-rules.yaml](../spec/business-rules.yaml) 为准
3. 接口骨架以 [openapi-structure.md](../spec/openapi-structure.md) 为准
4. 核心字段契约以 [api-contract-core.yaml](../spec/api-contract-core.yaml) 为准
5. 数据表以 [数据库表结构初稿.md](./数据库表结构初稿.md) 和 [01_lb_business_schema.sql](../ruoyi-vue-pro/sql/mysql/01_lb_business_schema.sql) 为准
6. 每完成一项任务，必须在本文档更新状态、完成日期、完成说明
7. 如果出现阻塞，也必须在本文档记录阻塞原因、时间、下一步处理动作

状态标记统一使用：

- `未开始`
- `进行中`
- `已完成`
- `阻塞`

## 4. 总体目标

项目最终上线标准不是“页面能打开”，而是以下目标全部成立：

1. 后端核心业务链路可真实运行
2. 管理端核心页面全部接入真实接口
3. 用户端与后端完成主链路联调
4. 支付、退款、提现、拆单、投诉申诉等高风险流程完成测试
5. 生产环境配置、部署、监控、备份、回滚方案齐全
6. 项目具备正式上线条件
