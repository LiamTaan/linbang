# 开发前检查清单

## 1. 使用场景

本清单用于每次正式开发前自检，目标是避免继续产出“死页面、假接口、字段对不上”的结果。

## 2. 开发前必查

### 2.1 仓库与框架

- [ ] 后端仓库确认是 `https://github.com/YunaiV/ruoyi-vue-pro`
- [ ] 管理端仓库确认是 `https://github.com/yudaocode/yudao-ui-admin-vue3`
- [ ] 没有误用 `yudao-boot-mini`
- [ ] 没有误用 `yudao-cloud-mini`

### 2.2 规则文件

- [ ] 已读取根目录 `AGENTS.md`
- [ ] 已读取 `spec/business-rules.yaml`
- [ ] 已读取 `spec/openapi-structure.md`
- [ ] 已读取 `spec/api-contract-core.yaml`
- [ ] 已阅读 `docs/邻里互助项目需求总稿.md`

### 2.3 业务对象

- [ ] 本次开发涉及的对象已在业务规则中存在
- [ ] 对象状态机已定义
- [ ] 对象权限边界已定义

### 2.4 接口契约

- [ ] 接口前缀已确认是 `/app-api` 或 `/admin-api`
- [ ] 请求字段已定
- [ ] 返回字段已定
- [ ] 分页结构已定
- [ ] 状态字典已定

### 2.5 页面交付

- [ ] 页面不是长期 mock 数据
- [ ] 页面操作有真实接口
- [ ] 页面状态与后端状态机一致
- [ ] 页面错误提示有处理
- [ ] 页面权限有处理

## 3. 资金相关必查

- [ ] 支付模块复用 `yudao-module-pay`
- [ ] 没有另造支付中心
- [ ] 支付回调幂等方案明确
- [ ] 退款回调幂等方案明确
- [ ] 钱包余额与流水关系明确
- [ ] 提现审核流明确

## 4. 拆单相关必查

- [ ] 拆分触发条件明确
- [ ] 单元金额上限明确
- [ ] 锁单逻辑明确
- [ ] 单元状态机明确
- [ ] 主订单与单元订单联动规则明确
- [ ] 人工解锁权限明确

## 5. 管理端必查

- [ ] 页面在当前版本内的实现优先级已明确
- [ ] 查询条件已定义
- [ ] 表格字段已定义
- [ ] 详情区块已定义
- [ ] 按钮动作已定义
- [ ] 操作日志要求已定义

## 6. 提交前必查

- [ ] 没有新增与规则冲突的命名
- [ ] 没有新增另一套状态值
- [ ] 没有自造新响应结构
- [ ] 没有只做 UI 壳子
- [ ] 文档、代码、接口三者一致

## 7. 当前阶段建议执行顺序

1. 先校验规则、表结构、接口契约三者一致
2. 再按模块实现后端真实接口
3. 再接管理端与用户端页面
4. 最后做联调与测试

如果真实接口没完成，不建议继续扩更多壳页面。
