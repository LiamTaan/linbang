# 邻里互助 Phase 0 模块清点

## 1. 目标

本清点用于把当前仓库中的 LinBang 模块按 `保留联调 / 改造升级 / 纯新增` 三标签固定下来，作为本轮开发的实施基线。

判定依据：

- `AGENTS.md`
- `spec/business-rules.yaml`
- `spec/openapi-structure.md`
- `spec/api-contract-core.yaml`
- `docs/邻里互助项目需求总稿.md`
- 当前 `ruoyi-vue-pro` 与 `yudao-ui-admin-vue3` 实际代码

## 2. 已有基础，可保留联调

以下模块在后端与管理端均已具备可复用底座：

- `member`
  - 用户、地址、实名、资质
- `merchant`
  - 服务商、入驻、服务类目、服务点
- `order`
  - 订单、拆分单元、抢单记录、匹配记录、异常订单、操作日志
- `wallet`
  - 钱包账户、银行卡、流水、提现、分账规则
- `review`
  - 投诉、申诉、评价、信用规则
- `risk`
  - 风控规则、敏感词

## 3. 已有骨架，但必须改造升级

- `auth`
  - 当前 LinBang App 认证门面仅完成短信验证码登录、退出、刷新令牌
  - 需升级为统一登录门面，复用芋道社交登录能力
- `dashboard`
  - 当前管理端首页只是模块跳转页，不是业务运营台
- `order`
  - 已有创建、分页、详情、接单、交付基础动作
  - 需继续补齐状态推进、锁单/解锁、退款进度、售后闭环
- `wallet`
  - 已有钱包详情、提现申请、绑卡
  - 需继续补齐记录分页、明细、冻结/托管口径
- `review`
  - 现有接口偏创建与基础审核
  - 需继续补齐处理进度、详情、信用联动

## 4. 当前基本缺失，需要新增

以下业务对象在当前 `dal/dataobject` 与 `controller` 中基本不存在，需要按强类型分层新增：

- `risk_event`
- `blacklist`
- `merchant_price_report`
- `promoter`
- `promoter_relation`
- `commission_order`
- `partner_info`
- `partner_region_rel`
- `message_template`
- `message_record`
- `dashboard` 聚合接口

## 5. 本轮实施范围

本轮优先完成：

1. Phase 0 契约与增量 SQL 资产
2. 缺失基础表与后端强类型对象
3. `dashboard / promote / partner / message / risk event / blacklist / price report` 的首批真实接口
4. 管理端首页从跳转页升级为真实运营台

本轮不直接推翻现有 `member / merchant / order / wallet / review / riskrule / sensitiveword` 页面结构。
